# 6-Sprint Backlog (2-week sprints)

Sprint 0: Planning & infra
- Finalize vision, success metrics, and target personas
- Complete brand/hero copy and IA for marketing site
- Wireframe landing + Formulator/editor flows
- Configure Vercel project, env vars, and NextAuth placeholders

Sprint 1: Public site + Content CMS
- Integrate Sanity studio (content types: hero, features, case studies)
- Connect marketing pages to Sanity content
- Add demo video and Calendly scheduling
- Deploy public site and enable analytics

Sprint 2: Formulator MVP (backend)
- Schema: formulas, ingredients, versioning
- Implement API endpoints: create/get/list formulas
- Ingredient search index (simple DB + seeded data)
- CSV import/export endpoints

Sprint 3: Formulator MVP (frontend)
- Basic formula editor UI (name, ingredients list, notes)
- Versioning UI and formula comparison
- Export lab-ready spec (PDF/CSV stub)
- Client-side validations and cost estimate stub

Sprint 4: Project Assistant MVP
- Backend: tasks, milestones, project summaries API
- Frontend: task list, create milestone, contextual chat stub
- Connect Project Assistant to formula records

Sprint 5: Sandbox, Auth, Integrations
- Complete NextAuth integration and sample GitHub OAuth app
- Sandbox tenant provisioning flow (demo tenants/seeds)
- Integrations: CSV import/export, webhook example (Stripe/webhook)
- QA, security review, E2E tests

Sprint 6: Launch & GTM
- Prepare sales enablement (pricing page, playbook, case studies)
- Run A/B tests on hero/CTA
- Setup onboarding email nurture + analytics dashboards
- Soft launch: pilot customers and collect feedback

Each backlog item should be turned into 1–3 concrete tickets with acceptance criteria.
